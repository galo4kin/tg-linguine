# Step 42.9 — refactor-groq-schema-retry-helper

## Откуда

Code review шага 40, finding §5: [_30_done/40-groq-respect-retry-after-review.md](_30_done/40-groq-respect-retry-after-review.md).

## Проблема

`Analyze` (`internal/llm/groq/analyze.go:78-102`) и `Adapt` (`internal/llm/groq/adapt.go:34-47`) делят почти идентичную обвязку:
1. валидация JSON через свою schema-функцию,
2. при провале — собрать retry-messages с assistant + user-instruction,
3. ещё один chat-вызов,
4. повторная валидация.

В шаге 39 в `Analyze` добавлен diagnostic-лог `groq.analyze schema-retry` со snippet/len. В `Adapt` такого лога нет — будущий «почему опять JSON в Adapt не валиден» придётся снова дебажить вслепую.

## Что делаем

1. Вытащить общий помощник в `internal/llm/groq/schema_retry.go`:
   ```go
   func (c *Client) chatJSONWithSchemaRetry(
       ctx context.Context,
       key, model string,
       messages []chatMessage,
       validate func([]byte) error,
       label string, // "groq.analyze" / "groq.adapt"
   ) ([]byte, error)
   ```
   Helper делает первый `c.chat`, валидирует, при провале логирует `<label> schema-retry` со snippet/len, вызывает `c.chat` второй раз, валидирует ещё раз и возвращает результат либо ошибку валидации.
2. `Analyze.Analyze` и `Client.Adapt` сворачиваются до подготовки messages + вызова helper'а.
3. Diagnostics распространяются на Adapt бесплатно.
