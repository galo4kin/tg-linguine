# Шаг 33. Стартовый пинг админу в личку (1h)

## Цель
Админ при каждом старте процесса получает в личку короткое
сообщение «бот поднялся» с версией и коммитом — чтобы было понятно,
что watchdog сработал и какая версия сейчас на проде.

## Контекст
Аналог в `tg-boltun`:
- `main.go:126-129`:
  `bot.Send(NewMessage(adminUserID, "🤖 Болтун проснулся ..."))`.

Опирается на `cfg.AdminUserID` из шага 31. Сообщение шлём только
если админ настроен.

`ADMIN_USER_ID` владельца бота — `1995215`. Это значение уже зашито в
`.env.example` (см. шаг 31), агент при реализации не должен спрашивать
его у пользователя.

## Действия
1. В `cmd/bot/main.go` после успешной инициализации бота и **только
   при** `cfg.AdminUserID != 0` отправить ОДНО сообщение в личку
   админа, например:
   ```
   🍝 tg-linguine поднялся
   version: <ver>
   commit: <hash>
   started: <time>
   ```
2. Источник `version` и `commit`:
   - Предпочтительно — `runtime/debug.ReadBuildInfo()` (без
     изменения Makefile).
   - Альтернатива — `-ldflags "-X main.version=... -X main.commit=
     $(git rev-parse --short HEAD)"` через `Makefile` цели `build`.
   Выбрать один способ при реализации.
3. Ошибка отправки сообщения **не считается фатальной** — только
   `slog.Warn("admin startup ping failed", "err", err)`. Бот
   продолжает работу.
4. На будущее (НЕ в этой задаче, но отметить в коде комментарием
   при необходимости): если появятся регулярные админ-уведомления
   (например, об ошибках LLM), стоит сделать throttled-хелпер по
   аналогии с `tg-boltun/instagram.go:376-416` (mutex + `lastNotify`
   + `notifyEvery`). Сейчас — только стартовый пинг.

## DoD
- Юнит-тест: при `cfg.AdminUserID == 0` хелпер `SendStartupPing` не
  обращается к Telegram API; при ненулевом — обращается с правильным
  `chat_id`, и текст содержит непустые `version` и `commit`.
- Без `ADMIN_USER_ID` ничего не отправляется и предупреждений в
  логах нет.
- Ошибка отправки логируется как `slog.Warn`, но не валит процесс.
- `make build` + `make test` зелёные.

Ручная проверка «реально пришло после ребута» делается одной строкой
в README-чек-листе после деплоя и не блокирует закрытие задачи.
