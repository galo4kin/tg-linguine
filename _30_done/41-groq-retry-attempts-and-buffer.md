# Step 41 — groq-retry-attempts-and-buffer

## Проблема

Шаг 40 уважает `Retry-After`, но первого ретрая не хватает: лог показывает summarize→analyze 429 (try again 4.85s) → ждём 5s → analyze 429 ещё раз (try again 4.795s). Free-tier TPM (12K/мин) скользящего окна недостаточно для summarize+analyze в одной минуте даже с одним ретраем.

## Что делаем

1. `chat()` и `chatPlainText()`: увеличить максимум попыток с **2 до 4** (исходный + 3 ретрая). Каждый ретрай уважает свой Retry-After hint.
2. К парсенному `parseRateLimitRetryAfter` добавлять небольшой **буфер 750 мс** перед сном — Groq иногда возвращает «4.795s», но к моменту перезапроса окно ещё не сместилось достаточно.
3. Прокинуть параметр `progress`-callback в Telegram-слой, чтобы пользователь видел `🧠 Анализирую…` пока бот спит между попытками. (Этот лог уже есть на стадии Analyze — достаточно, чтобы пользователь не думал, что бот завис.)
4. Юнит-тесты: правка существующих кейсов под буфер + новый «два 429 подряд → три ретрая → успех».
