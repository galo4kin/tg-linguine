## Найденное при ревью

- Тип `handlers.Study` оставлен с историческим названием, хотя по сути это quiz handler. Ниже типа FSM уже `*session.Quiz`, поле в `Delete` тоже. Единственная причина оставить — стабильный внешний контракт: `CallbackPrefixStudy = "study:"` нужен для входящего callback из `card.go` ("Show all words" → `study:start`) и `/study` команда. Если в будущем кто-то будет читать код без контекста — может зацепиться. Кандидат на чистый рефактор «переименовать в Quiz, сменить prefix на `quiz:` и обновить card.go» (можно вместе с одной из последующих задач 52–55, не отдельно).
- `card.go` использует `CallbackPrefixStudy + "start"` — это исторический артефакт «по клику на статью открыть карточки». Сейчас это «по клику открыть квиз». Семантика поведения соответствует новой фиче, но имя кнопки `article.card.show_all_words` не отражает того, что запускается квиз. Это вне скоупа задачи 50; можно учесть в шаге 55 (cleanup/help).
- `renderQuizQuestion` и `renderQuizFeedback` дублируют верхнюю часть рендера (заголовок + лемма/IPA/перевод). Дубликация на 6 строк — не критично, но при добавлении poll-варианта (шаг 51) станет 3-я копия. Если так — вынести `renderQuizPromptHeader(loc, snap, card)` (ок-кандидат на рефактор после шага 51).
- `Study.pickDirection` / `buildOptions` — тонкие mu-обёртки над session-хелперами. Один вызов на каждый. Inlining упростит чтение, но не критично.
- `dictionary.SampleDistractors` использует `ORDER BY RANDOM() LIMIT poolLimit`. На словарях в десятки тысяч строк это всё ещё OK для SQLite на маке, но если БД вырастет — стоит индекс или alternative sampling. Вне скоупа сейчас.
