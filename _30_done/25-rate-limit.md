# Шаг 25. Rate limiting (2–3h)

## Цель
Защита от случайной отправки 50 ссылок подряд: лимит 10 анализов в
час на пользователя.

## Действия
1. `internal/telegram/ratelimit.go`: in-memory token bucket
   per-user (map + mutex). Параметры: capacity=10, refill=1/6min.
2. Применяется только к URL-handler (шаг 12). На `/start`, settings
   и т.д. — нет.
3. При превышении — i18n-сообщение «Лимит N/час исчерпан, попробуйте
   через X минут».
4. Параметры лимита — в config (`RATE_LIMIT_PER_HOUR`, дефолт 10).

## DoD
- 11-й URL в течение часа → блокирующее сообщение.
- Через час слот освобождается.
- Сообщение содержит время до восстановления (приблизительно).
- `make build` + `make test` зелёные (юнит-тест на bucket).
