# Шаг 24. /delete_me и GDPR (3–4h)

## Цель
Полное удаление данных пользователя. `dictionary_words` остаются
(общий словарь).

## Действия
1. Команда `/delete_me`: бот спрашивает «Вы уверены? Это удалит
   все ваши данные» с inline-кнопками «Да, удалить» / «Отмена».
2. На подтверждение — транзакция, удаляющая записи пользователя из:
   `users`, `user_languages`, `user_api_keys`, `articles` (каскадно
   `article_words` по FK), `user_word_status`. `dictionary_words`
   не трогаем.
3. ON DELETE CASCADE на FK уже должны быть в миграциях; при
   необходимости добавить миграцию-исправление.
4. Сообщение об успехе; FSM-стейты сбросить.

## DoD
- После `/delete_me` ни в одной таблице нет записей пользователя
  (проверка SQL после теста).
- `dictionary_words` не уменьшается.
- Подтверждение обязательно — без него удаление не идёт.
- `make build` + `make test` зелёные.
