# Шаг 32. Админ-команды в боте (2–3h)

## Цель
Дать админу набор служебных команд в личке с ботом: посмотреть
статистику, корректно выключить процесс, проверить роль.

## Контекст
Опирается на `IsAdmin` из шага 31. Аналог в `tg-boltun`:
- Гейт по `userID != adminUserID` (`main.go:495`).
- Перезапуск через `os.Exit` (`main.go:404`) — watchdog поднимает.

Watchdog в нашем проекте создаётся на шаге 14 (`scripts/linguine-
watchdog.sh`), к моменту шага 32 он уже стоит в crontab.

## Действия
1. Регистрация команд в `internal/telegram/` с гейтом по
   `IsAdmin`. Если команду шлёт не-админ — игнор (ничего не
   отвечаем, чтобы не «светить» наличие админ-режима).
2. `/stats` — текстовый ответ:
   - всего пользователей: `SELECT COUNT(*) FROM users`,
   - активных за 24h / 7d (по `last_seen_at`; если поля нет —
     добавить миграцию и обновлять при каждом апдейте от юзера),
   - сколько статей проанализировано всего и за сутки,
   - сколько слов в словарях суммарно.
   Запросы — через соответствующие репозитории
   (`internal/users`, `internal/articles`, `internal/dictionary`).
3. `/shutdown` — graceful exit:
   - `slog.Warn("admin requested shutdown", ...)`,
   - остановить bot/poller,
   - `os.Exit(0)`.
   Watchdog поднимет в течение минуты.
4. `/whoami` — отвечает текстом «admin» / «user» + `user_id`. Нужно
   для smoke-проверки конфига после деплоя на mac mini.
5. i18n-строки только под ключом `admin.*` — RU + EN. Для не-админа
   ответов нет (см. п.1), так что строки нужны только для самих
   admin-ответов.

## DoD
- `/stats` от админа возвращает ответ; числа совпадают с прямыми
  SQL-запросами к `bot.db`.
- `/shutdown` от админа кладёт процесс; через ≤1 минуту watchdog
  поднимает его обратно.
- `/whoami` корректно различает админа и не-админа.
- Не-админ, отправивший `/stats`, `/shutdown` или `/whoami` без
  прав, не получает ответа и не валит бота.
- `make build` + `make test` зелёные.
