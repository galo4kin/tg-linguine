# Шаг 5. i18n (3–4h)

## Цель
UI на RU/EN/ES, выбор языка по `User.LanguageCode` с fallback на `en`.

## Контекст
Библиотека: `github.com/nicksnyder/go-i18n/v2/i18n`. Файлы локалей в
`/locales/{ru,en,es}.yaml`, встроены через `embed.FS`.

## Действия
1. `internal/i18n/bundle.go`: загрузка bundle, регистрация YAML
   unmarshal, embed-загрузка трёх файлов.
2. `internal/i18n/localizer.go`: `For(lang string) *i18n.Localizer` —
   нормализует код языка (`ru`, `en`, `es`), всё остальное → `en`.
3. Helper `T(loc, msgID, data) string` — обёртка с дефолтным fallback
   при отсутствии ключа.
4. Локали — стартовый набор:
   - `start.greeting` — приветствие из шага 4.
   - `error.generic` — общая ошибка.
5. Middleware в `internal/telegram/bot.go`: достаёт `update.Message.
   From.LanguageCode`, кладёт `*i18n.Localizer` в контекст.
6. Хендлер `/start` использует `T(loc, "start.greeting", nil)` вместо
   хардкода «Hello».

## DoD
- `/start` для пользователя с `language_code=ru` → русский текст.
- `language_code=es` → испанский.
- `language_code=de` (или любой другой) → английский.
- Отсутствующий ключ → возвращается ID ключа + лог Warn.
- `make build` зелёный.

## Зависимости
Шаги 1, 4.
