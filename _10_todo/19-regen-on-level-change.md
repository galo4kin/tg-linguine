# Шаг 19. Регенерация адаптации при смене уровня (3–4h)

## Цель
Если пользователь сменил CEFR — открытая старая статья догенерирует
недостающую адаптацию.

## Действия
1. При запросе адаптации проверить, есть ли она в
   `adapted_versions` (json) для нужного уровня.
2. Если нет — частичный LLM-вызов: специальный промпт «адаптируй
   текст под уровень X», без полного анализа. Контракт ответа —
   мини-schema {`adapted_text`, `summary_target`}.
3. Сохранить в `adapted_versions` (merge JSON).
4. UI: пока идёт генерация — статус «Адаптирую под <level>…», потом
   обновлённое сообщение.

## DoD
- Меняем CEFR в settings (шаг 20 ставит UI; здесь работаем напрямую
  через БД, если settings ещё нет — допустимо), открываем старую
  статью → недостающая адаптация догенерируется и кэшируется.
- Повторный запрос той же адаптации — без LLM (cache hit).
- `make build` + `make test` зелёные.
