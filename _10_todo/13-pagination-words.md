# Шаг 13. Просмотр всех слов из статьи (3–4h)

## Цель
Inline-пагинация по 5 слов с переводом, транскрипцией и примером.

## Контекст
Финальный шаг Walking Skeleton. После него бот пригоден к ручному
тестированию.

## Действия
1. Callback-handler `words:<article_id>:<page>`.
2. Запрос к `article_words` JOIN `dictionary_words` с offset/limit.
3. Формат сообщения для каждого слова:
   ```
   • <surface_form> (<lemma>, <pos>) [<ipa>]
     → <translation_native>
     <example_target>
     <example_native>
   ```
4. Кнопки навигации: «◀ Prev», «Next ▶», «Закрыть». Disable, если
   страница первая/последняя.
5. i18n-строки для подписи, кнопок, «Слов нет».

## DoD
- Из article card можно открыть полный список слов.
- Постраничный просмотр работает в обе стороны.
- На статье с N словами — `ceil(N/5)` страниц.
- `make build` + `make test` зелёные.

## Этап
🎯 Конец Фазы 1 (Walking Skeleton). После этого шага можно деплоить.
