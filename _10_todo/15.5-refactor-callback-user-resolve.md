# Рефактор. Общий хелпер для регистрации пользователя в хендлерах (1h)

## Цель
Убрать дублирование 4–5-строчного блока `users.RegisterUser` из всех
callback- и message-хендлеров.

## Контекст
Сейчас одна и та же конструкция повторяется в:
- `internal/telegram/handlers/onboarding.go` — `HandleLanguage`,
  `HandleLevel` (по разу);
- `internal/telegram/handlers/words.go` — внутри `resolveUser`
  (приватного метода);
- `internal/telegram/handlers/url.go` — `Handle` (один раз);
- `internal/telegram/handlers/apikey.go` — `HandleSetKeyCommand`,
  `HandleIncomingText` (по разу).

Каждое место строит `users.TelegramUser{ID, Username, FirstName,
LanguageCode}`, вызывает `users.Service.RegisterUser`, игнорирует
`created`, логирует ошибку и возвращается. Это шум, и у Words мы
уже вынесли локальную копию в `(*Words).resolveUser`.

## Действия
1. В `internal/telegram/handlers/` создать новый файл `helpers.go`
   (или дополнить существующий) с двумя свободными функциями:
   ```go
   func resolveCallbackUser(ctx context.Context, svc *users.Service,
       cq *models.CallbackQuery, log *slog.Logger,
   ) (*users.User, bool)

   func resolveMessageUser(ctx context.Context, svc *users.Service,
       msg *models.Message, log *slog.Logger,
   ) (*users.User, bool)
   ```
   Каждая функция: формирует `users.TelegramUser`, вызывает
   `RegisterUser`, при ошибке логирует с понятным префиксом и
   возвращает `nil, false`.

2. Переписать использования в onboarding/words/url/apikey на эти
   хелперы. У `Words` удалить приватный `resolveUser` и пользоваться
   общим хелпером (он также возвращает локалайзер? — оставь
   локализацию вызывающей стороне через `tgi18n.For(bundle,
   u.InterfaceLanguage)`, чтобы хелпер не тянул bundle).

3. Прогнать `make build` и `make test`.

## DoD
- В пакете `handlers` есть один (или два) общий хелпер для извлечения
  пользователя из update.
- Все 6+ дублирующих мест используют этот хелпер.
- `make build` и `make test` зелёные.
- Поведение хендлеров не изменилось (логирование может стать чуть
  единообразнее).
