# Step 42.8 — refactor-groq-tpm-constants-colocate

## Откуда

Code review шага 40, finding §4: [_30_done/40-groq-respect-retry-after-review.md](_30_done/40-groq-respect-retry-after-review.md).

## Проблема

Константы, описывающие Groq free-tier бюджет, размазаны:
- `internal/articles/usecase.go`: `DefaultMaxTokensPerArticle = 5000`, `summarizeInputBudget = 7500`.
- `internal/llm/groq/analyze.go`: `analyzeMaxCompletionTokens = 4000`.
- `internal/llm/groq/client.go`: `maxRetryAfter = 60s`, `retryAfterBuffer = 750ms`, `maxRateLimitAttempts = 4`.

Связь между числами (5K + 4K + ~1K + summary = ≤12K TPM) держится только в комментариях разных файлов — изменить один лимит и не сломать суммарную сходимость стало ручной работой.

## Что делаем

1. Создать `internal/llm/groq/budget.go` с константами Groq-бюджета и общим комментарием, объясняющим связь:
   ```go
   // Free-tier TPM budget (~12K per request, input + reserved output).
   // ArticleAnalyzeOutputCap + DefaultArticleInputCap + ~1K system prompt
   // must stay under TPM. Increase together when moving to a paid tier.
   const (
       FreeTierTPM = 12000

       ArticleAnalyzeOutputCap = 4000
       DefaultArticleInputCap  = 5000

       SummarizeInputCap = 7500
       // SummarizeOutputCap is computed at call site as TargetTokens+500.

       MaxRetryAfter        = 60 * time.Second
       RetryAfterBuffer     = 750 * time.Millisecond
       MaxRateLimitAttempts = 4
   )
   ```
2. Удалить дубликаты из `analyze.go`, `client.go`, `articles/usecase.go`. `articles/usecase.go` берёт `DefaultArticleInputCap` через импорт `groq` пакета (или re-export через `articles`-типа `DefaultMaxTokensPerArticle = groq.DefaultArticleInputCap`).
3. `summarizeInputBudget` сохраняется как алиас или просто исчезает, везде используем `groq.SummarizeInputCap`.
4. README — уточнить ссылку «12K TPM cap», если упоминается.
